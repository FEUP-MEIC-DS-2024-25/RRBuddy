REQUISITOS FUNCIONAIS

Funcionalidade do Sistema
- Um usuário deve ser capaz de redefinir sua senha por meio de um processo de verificação por email.
- Um gerente deve ser capaz de gerar relatórios de vendas mensais em formato PDF.
- Os clientes devem ser capazes de rastrear o status de seus pedidos em tempo real.
- Um administrador deve ser capaz de criar, atualizar e excluir contas de usuários.
- A plataforma deve oferecer suporte à integração com gateways de pagamento de terceiros, como PayPal e Stripe.
- Uma pesquisa de produto deve retornar resultados relevantes em até 2 segundos após a entrada.
- Os usuários devem ter a capacidade de enviar arquivos de até 50 MB de tamanho.
- O sistema deve manter um log de todas as atividades do usuário para fins de auditoria.


Funcionalidade de Notificação
- Notificações devem ser enviadas aos usuários 24 horas antes do vencimento de sua assinatura.
- Os usuários devem receber um alerta se tentarem excluir um item permanentemente.


REQUISITOS NÃO FUNCIONAIS

Desempenho
- O sistema deve suportar 10.000 usuários simultâneos sem degradação de desempenho.
- O aplicativo deve carregar o painel em 3 segundos em condições normais.
- Uma pesquisa de produto deve retornar resultados relevantes em até 2 segundos após a entrada.
- O sistema deve se recuperar de quedas de energia em 1 minuto sem perder dados.
- O software deve estar operacional por pelo menos 99,9% do tempo a cada mês.


Segurança
- Todos os dados devem ser criptografados tanto em trânsito quanto em repouso.
- A plataforma deve suportar autenticação multifator para login.


Usabilidade
- A interface do usuário deve suportar os idiomas inglês, espanhol e francês.
- O portal da web deve atender aos padrões de acessibilidade WCAG 2.1 AA.


Compatibilidade
- O aplicativo móvel deve ser compatível com iOS 14 e posterior.
- O software deve ser implantável em ambientes baseados em nuvem e locais.


Conformidade e Manutenção
- Os dados do cliente devem ser armazenados por pelo menos cinco anos para cumprir as regulamentações legais.
- O sistema deve fornecer um recurso de helpdesk online 24 horas por dia, 7 dias por semana para os usuários.
- Relatórios devem incluir gráficos visuais para resumir as tendências de dados.
- Desenvolvedores devem usar Python 3.10 ou posterior para serviços de back-end.
